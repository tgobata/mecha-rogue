# ドキュメント更新ガイド

敵・ボス・アイテム・主人公・ショップのデータを追加・修正した後に、`docs/` 配下の HTML ファイルを最新状態に保つ手順を説明します。

---

## 概要

`docs/` の HTML ファイルは、`src/game/assets/data/` の JSON データを `<script>` タグに直接埋め込む形式になっています。
JSON を fetch する代わりにインライン埋め込みにしている理由は、ファイルをブラウザでローカル直接開いたときの **CORS エラーを回避するため**です。

**JSON を編集しただけでは HTML は自動更新されません。** 以下のコマンドを実行して同期してください。

---

## 更新スクリプト一覧

| スクリプト | 対象 HTML | 読み込む JSON |
|---|---|---|
| `scripts/embed-json-in-docs.js` | 全敵 / 全ボス / 全アイテム / 主人公 / ショップ | enemies / bosses / items / weapons / tools-equipment / shop-prices |
| `scripts/update-enemy-docs.ts` | `docs/全敵.html` のみ | `enemies.json` |
| `scripts/update-boss-docs.ts`  | `docs/全ボス.html` のみ | `bosses.json` |

通常は **`embed-json-in-docs.js` を 1 回実行するだけ**で 5 ファイルすべて更新されます。

---

## 実行方法

プロジェクトルートで実行します。

```sh
# 5ファイルまとめて更新（推奨）
node scripts/embed-json-in-docs.js
```

成功すると以下のようなメッセージが出力されます。

```
全敵.html: 書き込み完了 (XXXXX bytes)
全ボス.html: 書き込み完了 (XXXXX bytes)
全アイテム.html: 書き込み完了 (XXXXX bytes)
主人公.html: 書き込み完了 (XXXXX bytes)
ショップ.html: 書き込み完了 (XXXXX bytes)

既存5ファイルの修正完了。
```

### 個別に更新する場合

```sh
# 敵だけ更新
node --experimental-strip-types scripts/update-enemy-docs.ts

# ボスだけ更新
node --experimental-strip-types scripts/update-boss-docs.ts
```

## 作業フロー（敵を新規追加する場合）

```
1. src/game/assets/data/enemies.json に新しい敵を追記
2. 対応するスプライト生成スクリプトを実行 (generate-sprites.ts 等)
   → public/sprites/enemies/ に PNG が生成される
3. node scripts/embed-json-in-docs.js を実行
   → docs/全敵.html・docs/全スプライト.html に反映される
4. ブラウザで docs/全敵.html と docs/全スプライト.html を開いて確認
```

同様に、ボスは `bosses.json`、アイテムは `items.json` / `weapons.json` / `tools-equipment.json` を編集後に手順 3 を実行します。

## トラブルシューティング

### 「置換パターンが見つかりませんでした」と表示される

HTML 内の `const ENEMIES_DATA = [...];` などのパターンが壊れている可能性があります。
該当 HTML を開き、`<script>` ブロックの構文を確認してください。

### スプライトが HTML に表示されない

`docs/全スプライト.html` は `../public/sprites/` への相対パスで PNG を参照しています。
ブラウザのローカルファイルセキュリティポリシーにより、直接開くと画像が表示されないブラウザがあります。
`npm run dev` でローカルサーバーを起動し、`http://localhost:3000` 経由で確認するか、VS Code の Live Server 拡張を使用してください。

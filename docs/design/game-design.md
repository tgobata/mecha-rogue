# 🎮 メカローグ ー 機甲迷宮探索記 ー

## ゲームデザインドキュメント（完全版）

---

## 1. ゲーム概要

### 1.1 コンセプト
プレイヤーは「メカマシン」に搭乗し、ランダム生成される多層迷宮を探索するターン制2Dローグライクゲーム。階層を降りるごとに難易度が上昇し、個性豊かなボスたちが待ち受ける。武器・道具・マシンの強化を繰り返しながら、最深部を目指す。

### 1.2 技術仕様
- **言語**: JavaScript（React単一ファイル .jsx）
- **実行環境**: ブラウザ（PC / スマートフォン対応）
- **描画**: Canvas 2D または DOM/CSS Grid ベースのタイル描画
- **データ保存**: `window.storage`（永続ストレージAPI）によるオートセーブ
- **入力**: PC→キーボード（WASD/矢印キー）＋マウス、スマホ→タッチ操作（仮想十字キー＋ボタン）
- **画面**: レスポンシブ対応、縦画面・横画面両対応

### 1.3 ゲームの流れ（コアループ）
```
探索 → 戦闘 → アイテム取得 → マシン強化 → 階段発見 → 次階層へ
  ↑                                                    ↓
  ← ← ← 撃破（スタート帰還）← ← ← ← ← ← ← ← ← ← ←
```

## 2. 迷路生成システム

### 2.1 生成アルゴリズム
- **基本手法**: 再帰分割法（Recursive Division）またはランダムPrimのアルゴリズムを使用
- **マップサイズ**: 階層に応じて拡大
  - 1〜5階: 25×25 タイル
  - 6〜10階: 30×30 タイル
  - 11〜20階: 35×35 タイル
  - 21〜30階: 40×40 タイル
  - 31階〜: 45×45 タイル
- **タイルサイズ**: 32×32px（スマホでは自動スケール）

### 2.2 マップ構成要素
| 記号 | 要素 | 説明 |
|------|------|------|
| `.` | 通路 | 移動可能な空間 |
| `#` | 壁 | 移動不可。攻撃で一部破壊可能（ひび割れ壁） |
| `>` | 下り階段 | 次の階へ進む。各階に1つだけ |
| `S` | スタート地点 | 各階の開始位置 |
| `$` | アイテム | ランダム配置 |
| `G` | お金 | ランダム配置 |
| `E` | 敵 | ランダム配置 |
| `B` | ボス | 特定階のボス部屋に配置 |
| `T` | 落とし穴 | 隠れている場合と可視の場合あり |
| `R` | 休憩ポイント | HP回復スポット |
| `W` | 武器修理ポイント | 耐久度回復スポット |
| `P` | ショップ | 売買可能 |
| `H` | ヒント石碑 | ボスの弱点情報 |
| `?` | 未発見タイル | 視界外（探索前の暗闇） |

### 2.3 視界システム（戦略の核）
- **視界範囲**: マシンから半径3タイル（デフォルト）
- **暗闇**: 未探索エリアは黒塗り、探索済みだが視界外はグレー表示
- **視界拡張アイテム**: 「スカウトレンズ」装備で半径5タイルに拡張
- **マップアイテム**: 「フロアマップ」使用でそのフロア全体が可視化される（敵位置は含まない）
- **敵センサー**: 「レーダーモジュール」装備で視界外の敵をレーダー表示

### 2.4 特殊地形
- **ひび割れ壁**: 攻撃で破壊可能。隠し通路やアイテム部屋への入口
- **水たまり**: 移動速度-1。電撃系武器のダメージ2倍
- **溶岩タイル**: 通過時にマシンHP -5（耐熱装甲で無効化可能）（10階以降に出現）
- **氷タイル**: 移動時に1タイル滑る（スパイクタイヤで無効化可能）（15階以降に出現）
- **ワープタイル**: 同フロア内の別のワープタイルに瞬間移動（20階以降に出現）
- **磁場タイル**: ミサイル・誘導弾が使用不可になるエリア（25階以降に出現）

### 2.5 部屋タイプ（迷路内にランダム生成）
- **モンスターハウス**: 入室すると四方から大量の敵が出現（高報酬）
- **宝物部屋**: 入口が隠し壁の奥にあり、レアアイテムが複数存在
- **罠部屋**: 落とし穴や地雷が大量に設置されている
- **ボス部屋**: ボスのいる大きな部屋。入室するとドアがロックされ、撃破まで出られない

## 3. 主人公とマシンシステム

### 3.1 パイロット（主人公）
主人公自身のステータスはシンプルに保つ。強さはマシンと装備に依存。

| パラメータ | 初期値 | 説明 |
|-----------|-------|------|
| パイロットLv | 1 | 経験値で上昇。スキルポイント獲得 |
| 操縦技術 | 10 | 回避率に影響。Lvアップで+1 |
| 判断力 | 10 | クリティカル率に影響。Lvアップで+1 |
| 幸運 | 5 | アイテムドロップ率に影響。Lvアップで+0.5 |

|---------|------|-------|
| 鉄壁操縦 | 被ダメージ-2%/Lv | 10 |
| ホークアイ | 視界範囲+0.5タイル/Lv | 4 |
| トレジャーハンター | アイテムドロップ率+3%/Lv | 10 |
| エンジニア | 武器耐久度消耗-5%/Lv | 10 |
| スピードスター | 移動速度+1（Lv5到達時のみ） | 5 |
| 値切り上手 | ショップ割引+3%/Lv | 10 |
| サバイバル | 休憩ポイントHP回復量+10%/Lv | 5 |

### 3.5 マシン撃破（HP0）時のペナルティ
- スタート地点（1階ロビー）に強制帰還
- **所持金**: 50%失う
- **所持アイテム**: ランダムで30〜50%消滅
- **装備中の武器**: 全て消滅
- **装備中の道具**: 全て消滅
- **マシン強化**: 維持される（永続強化は失われない）
- **倉庫内の物品**: 全て安全（影響なし）
- **パイロットLv・スキル**: 維持される

### 4.2 武器の耐久度システム
- 攻撃1回につき耐久度 -1（一部武器は -2 や -0.5）
- 耐久度が最大値の25%以下で攻撃力が 70% に低下
- 耐久度が0になると武器は破壊され消滅
- **修理方法**:
  - 武器修理ポイント（5階ごと）: 耐久度を最大値の50%まで無料回復
  - 修理キット（アイテム）: 耐久度を最大値まで完全回復
  - ショップ修理: お金を払って完全回復（武器のレアリティで価格変動）

### 4.3 武器のレアリティ
同じ武器でもレアリティがあり、高レアリティほどステータスにボーナスが付く。

| レアリティ | 色 | ボーナス | 出現確率 |
|-----------|-----|---------|---------|
| コモン（C） | 白 | なし | 50% |
| アンコモン（U） | 緑 | 攻撃力+10%, 耐久度+10% | 28% |
| レア（R） | 青 | 攻撃力+25%, 耐久度+20%, 特殊効果付与率あり | 15% |
| エピック（E） | 紫 | 攻撃力+40%, 耐久度+30%, 特殊効果確定 | 5.5% |
| レジェンド（L） | 金 | 攻撃力+60%, 耐久度+50%, 固有特殊効果 | 1.5% |

### 4.4 武器の特殊効果（レア以上でランダム付与）
- **炎上**: 攻撃時10%で3ターン継続ダメージ（毎ターン攻撃力の10%）
- **凍結**: 攻撃時8%で1ターン行動不能
- **感電**: 攻撃時12%で水たまり上の敵に2倍ダメージ
- **吸血**: 与ダメージの10%をHP回復
- **貫通**: 装甲値を50%無視
- **連撃**: 20%の確率で同ターン内に2回攻撃

### 4.5 武器強化
特定アイテム「強化コア」を消費して武器をレベルアップできる。
- 武器Lv1→2: 強化コアI × 3
- 武器Lv2→3: 強化コアII × 3
- 武器Lv3→4: 強化コアIII × 5
- 武器Lv4→5（最大）: 強化コアIII × 10
- レベルアップごとに攻撃力+8%、耐久度最大値+5

## 6. アイテムシステム

### 6.1 アイテム分類

| カテゴリ | 例 |
|---------|-----|
| 消耗品 | リペアキット、エネルギーパック等（上記道具の使い捨て型と共通） |
| 強化パーツ | マシンのステータスを永続強化する素材 |
| 強化コア | 武器のレベルアップに使用する素材 |
| 倉庫拡張チケット | 倉庫容量を増やす |
| 素材アイテム | ショップでの売却や交換用 |
| 鑑定前アイテム | 「？？？のパーツ」等。鑑定すると正体判明（良い物もハズレもある） |
| イベントアイテム | ボスの弱点アイテム、クエスト関連 |

### 6.2 アイテムドロップの仕組み
- **フロアドロップ**: 各フロアにランダムで 3〜8 個のアイテム・お金が配置
- **敵ドロップ**: 敵撃破時に 60% の確率でアイテムまたはお金をドロップ
- **ボスドロップ**: 100% で高レアリティアイテム＋大量のお金＋確定で強化素材
- **階層補正**: 深い階ほど高レアリティのアイテムが出やすい
  - 1〜5階: コモン 70%, アンコモン 25%, レア 5%
  - 6〜10階: コモン 50%, アンコモン 30%, レア 15%, エピック 5%
  - 11〜20階: コモン 30%, アンコモン 30%, レア 25%, エピック 12%, レジェンド 3%
  - 21〜30階: コモン 15%, アンコモン 25%, レア 30%, エピック 20%, レジェンド 10%
  - 31階〜: コモン 5%, アンコモン 20%, レア 30%, エピック 28%, レジェンド 17%

### 6.3 お金（ゴールド）
- 敵撃破、フロアドロップ、アイテム売却で入手
- ショップでの購入、倉庫拡張、武器修理等に使用
- 深い階ほどドロップ額が増加（1階あたり基本値 × 1.15^階層 で増加）

## 13. ターンシステム詳細

### 13.1 基本ルール
- プレイヤーが1回行動 → 全ての敵が1回行動（これが1ターン）
- プレイヤー行動 = 移動1回 or 攻撃1回 or アイテム使用1回 or 待機
- 移動速度2以上の場合: 1ターンで2タイル以上移動可能（途中で敵に隣接すると停止選択可）

### 13.2 行動速度の処理
- 移動速度2のプレイヤー: 1ターンで2タイル移動可能
- 速度3のボス: プレイヤーの1ターンに対し3回行動
- 処理順序: プレイヤー行動 → 敵1の行動 → 敵2の行動 → ... → ターン終了

### 13.3 状態異常

| 状態異常 | 効果 | ターン数 |
|---------|------|---------|
| 凍結 | 行動不能。受けるダメージ1.5倍 | 1〜3 |
| 感電 | 移動不能（攻撃は可能） | 2 |
| 炎上 | 毎ターン最大HPの5%ダメージ | 3 |
| 暗闇 | 視界1タイルに制限 | 5 |
| 混乱 | 移動方向がランダム | 3 |
| 行動不能(EMP) | 一切の行動不可 | 2〜3 |
| スロウ | 移動速度1固定 | 4 |

## 15. バランス調整の指針

### 15.1 難易度カーブ
- 1〜5階: チュートリアル相当。基本操作と仕組みを学ぶ
- 6〜10階: 装備・強化の重要性を実感。初の上級ボスで壁を感じる
- 11〜20階: 中級者向け。特殊地形・状態異常・道具の戦略運用が必要
- 21〜30階: 上級者向け。武器の選択とボス弱点の知識が必須
- 31〜40階: ハードコア。全てのシステムを使いこなさないと突破不可能
- 41〜50階: エンドコンテンツ。最高の装備と知識と戦略が要求される

### 15.2 ダメージ計算式
```
最終ダメージ = (武器攻撃力 × 武器レベル補正 × レアリティ補正 × バフ補正)
               - (対象の装甲値 × (1 - 貫通率))
最小ダメージ = 1（常に最低1ダメージは入る）
クリティカル = 最終ダメージ × 1.8（判断力で発生率変動、基本5%）
```

### 15.3 敵HP・攻撃力のスケーリング
```
敵の基本HP × (1.12 ^ 現在階層)
敵の基本攻撃力 × (1.10 ^ 現在階層)
```

## 17. 技術メモ

### 17.1 表示方式の選択肢
- **Option A: CSS Grid + 絵文字/Unicode**: 実装が最もシンプル。`@`=プレイヤー、`E`=敵 等
- **Option B: Canvas 2D**: パフォーマンス良好。スプライトシートで描画
- **Option C: React + DOM**: Reactの状態管理と相性が良い。Tailwind CSSで装飾

推奨: **Option C**（React .jsx 単一ファイル）で開始し、パフォーマンスが必要ならCanvasに部分移行

### 17.2 ストレージ設計
```javascript
// 1つのキーに全セーブデータを格納（ストレージAPI呼び出し最小化）
await window.storage.set('roguelike-save', JSON.stringify(fullGameState));
const result = await window.storage.get('roguelike-save');
const gameState = JSON.parse(result.value);
```

### 17.3 迷路生成の疑似コード
```
function generateMaze(width, height):
  1. 全タイルを壁で初期化
  2. ランダムな開始点から再帰的に壁を掘る（DFS）
  3. 部屋をランダムに3〜6個配置（大きさ3×3〜6×6）
  4. 部屋同士が通路で接続されていることを保証
  5. 階段・アイテム・敵・特殊タイルを配置
  6. スタート地点から階段まで到達可能であることを検証
```

### 17.4 パフォーマンス考慮
- 描画はプレイヤー周辺のタイルのみ（ビューポートカリング）
- 敵AIは視界外でも計算するが、アニメーションは省略
- 大きなマップ（45×45）でも2D配列なら十分高速
